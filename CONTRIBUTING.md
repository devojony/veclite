# 贡献指南

感谢您对 Veclite 项目的关注！我们欢迎任何形式的贡献。

## 如何贡献

### 报告问题

如果您发现了 bug 或有功能建议，请：

1. 在 Issues 中搜索是否已有相关问题
2. 如果没有，创建一个新的 Issue
3. 清楚地描述问题或建议，包括：
   - 问题的详细描述
   - 复现步骤（如果是 bug）
   - 预期行为
   - 实际行为
   - 环境信息（Android 版本、设备型号等）

### 提交代码

1. Fork 本仓库
2. 创建您的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交您的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启一个 Pull Request

### 代码规范

- 遵循 Kotlin 官方代码风格指南
- 确保代码可以正常编译
- 添加必要的注释
- 保持代码简洁清晰

### Pull Request 指南

- 确保 PR 描述清楚地说明了更改的内容和原因
- 关联相关的 Issue（如果有）
- 确保所有测试通过
- 保持提交历史清晰

## 开发环境设置

1. 克隆仓库
2. 使用 Android Studio 打开项目
3. 同步 Gradle 依赖
4. 运行项目

## 许可证

通过贡献代码，您同意您的贡献将在 MIT 许可证下发布。
